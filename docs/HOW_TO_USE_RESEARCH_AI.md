# 研究AI 使い方ガイド

## 🎯 どこから使えるの？

### アクセス方法

```
トップページ
  ↓
レース一覧ページ
  ↓
レース詳細ページ  ← ここに「🔬 研究AI」タブがある
```

---

## 📍 具体的な手順

### 1. レース一覧ページにアクセス

**URL形式**:
```
https://your-app.railway.app/races/YYYYMMDD
```

**例**:
```
https://your-app.railway.app/races/20260118
```

このページで、その日のレース一覧が表示されます。

---

### 2. 分析したいレースをクリック

レース一覧から任意のレースをクリックすると、レース詳細ページに移動します。

**URL形式**:
```
https://your-app.railway.app/race/YYYYMMDDCCNN
```

**例**:
```
https://your-app.railway.app/race/202601180611
                                    ↑     ↑ ↑
                                  日付   場 R
```

- `20260118`: 2026年1月18日
- `06`: 中山競馬場
- `11`: 11R

---

### 3. 「🔬 研究AI」タブをクリック

レース詳細ページには2つのタブがあります：

```
┌─────────┬──────────┐
│ 📊 出走表 │ 🔬 研究AI │  ← このタブをクリック
└─────────┴──────────┘
```

- **📊 出走表**: 通常の出走馬一覧
- **🔬 研究AI**: AI分析機能（新機能）

---

### 4. 研究を開始

**入力欄**:

1. **ゴール欄**（必須）
   ```
   例: 過大評価の理由を明らかにする
   例: 低人気で期待値のある馬を見つける
   例: 血統的に有利な馬を探す
   ```

2. **疑問欄**（必須）
   ```
   例: 5番はなぜ人気なのか？
   例: 7番の馬について調べて
   例: このレースで血統が良い馬は？
   ```

3. **「研究開始」ボタン**をクリック

---

### 5. 結果を確認

AIが自動で複数の分析を実行し、リアルタイムで表示されます：

```
━━━━━━━━━━━━━━━━━━━━━
```

## 💡 使用例

### 例1: 人気馬の分析

```
ゴール: 1番人気の過大評価の理由を調べる
疑問: 1番人気の馬について調べて
```

**AIが実行する分析**:
1. 血統分析 → 種牡馬の適性をチェック
2. 母父分析 → 母系の影響を確認
3. コース分析 → このコースでの実績
4. レベル分析 → 前走の質を評価

### 例2: 穴馬の発掘

```
ゴール: 低人気で期待値のある馬を見つける
疑問: 7番の馬はどうか
```

**AIが実行する分析**:
1. 血統分析 → 距離適性
2. 枠順分析 → 枠の有利不利
3. タイム補正 → 真の実力
4. オッズ分析 → 期待値の検証

### 例3: 条件を絞る

```
ゴール: 母父が強い馬を探す
疑問: 母父について調べて
```

**AIが実行する分析**:
1. 母父分析 → 母父の影響を調査
2. 血統分析 → 父との相性（ニックス）
3. コース分析 → コース実績

## ❓ トラブルシューティング

### Q1: 「研究AI」タブが見つからない

**確認事項**:
- デプロイ済みか？（`git push` 完了後、Railwayで自動デプロイ）
- キャッシュをクリアしてリロード（Ctrl + F5）

### Q2: エラーが出る

**よくあるエラー**:

```
エラー: Unauthorized - Premium required
```
→ プレミアム会員チェックが有効。`app/api/research/route.ts` で一時的に無効化可能。

```
エラー: OpenAI API key not found
```
→ 環境変数 `OPENAI_API_KEY` が未設定。Railway の環境変数に追加。

### Q3: 結果が出ない

**確認事項**:
- umadata テーブルにデータがあるか？
- 馬名が正確に入力されているか？
- OpenAI APIの利用制限に引っかかっていないか？

## 🔧 開発者向け情報

### ファイル構成
```
app/
  race/[raceKey]/page.tsx        # タブ切り替え実装
  components/ResearchPanel.tsx   # 研究UIコンポーネント
  api/research/route.ts          # 研究APIエンドポイント
  api/ai-tools/*/route.ts        # 各分析ツール

lib/research/
  research-engine.ts             # AI実行エンジン
  tools-registry.ts              # ツール定義
  performance-calculator.ts      # 期待値評価
```

### デバッグ方法
```bash
# ローカルで確認
npm run dev

# ブラウザのコンソールでエラー確認
F12 → Console タブ

# サーバーログ確認（Railway）
Railway Dashboard → Deployments → Logs
```
