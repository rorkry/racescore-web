## 1. L4F の意味と方向【確定】

## **L4F = 後半4F（800m）のラップ秒。小さいほど速い。**

### 根拠

| 検定 | 結果 | 判定 |
|---|---:|---|
| **corr(L4F, last_3f)** Pearson | **+0.9836** | ほぼ同一の測定量。L4F は「秒」 |
| corr(L4F, last_3f) Spearman | +0.9804 | 同上 |
| レース内 corr(L4F順位%, last_3f順位%) | **+1.0000** | 完全一致 |
| レース内 corr(L4F順位%, 着順%) | **+0.5378** | L4F が小さい馬ほど好着順 |

**決定的な裏付け（1F分の差）:**

| 馬場 | L4F 中央値 | last_3f 中央値 | 差 |
|---|---:|---:|---:|
| 芝 | 47.30 秒 | 35.50 秒 | **11.80 秒** |
| ダート | 51.30 秒 | 39.00 秒 | **12.30 秒** |

差はちょうど 1F（200m）分の走破時間。**L4F は last_3f に1F足した区間の秒数**であることが確定した。

**四分位群比較（n=21,999）:**

| 群 | n | 平均 last_3f | 平均着順 |
|---|---:|---:|---:|
| L4F 小さい群（≤47.40） | 5,548 | **34.752** | **12.538** |
| L4F 大きい群（≥51.80） | 5,511 | 41.192 | 13.046 |

芝・ダート個別でも corr(L4F, last_3f) = 0.9648 / 0.9726 で符号は同一。**芝ダで意味は変わらない**（水準だけ違う）。

### legacy への影響（修正しない・v2で正す）

`lib/race-simulator/capability-analyzer.ts:194-215` は L4F を **「大きいほど良い」** として
`acceleration` に 60% で反映している（`L4F>=50 → 95点`, `L4F=42 → 30点`）。

**これは方向が完全に反転している。** 50秒は遅く、42秒は速い。
つまり legacy は「後半が遅い馬」に高い acceleration を与えていた。

ただし監査で判明したとおり `score/weight*100` の飽和で全馬 100 になるため、
**この反転は最終着順には現れていない**（飽和がバグを隠していた）。
legacy は方針どおり修正しない。v2 では `normalizeLowerIsBetter(L4F)` を用いる。

### v2 での扱い

方向が確定したため **L4F を有効な後半指標として採用する**。
ただし後半モデルの主軸は充足率の高い `last_3f`（99.2%）とし、L4F（93.9%）は補助に置く。
設定 `L4fDirection` で `'disabled'` に切り替えられる構造を残す。

## 2. finish_position は全角/半角混在【重大な罠】

`finish_position` の値分布（全822k行）:

| 値 | 件数 | 表記 |
|---|---:|---|
| `１`〜`９` | 各 55,737〜58,717 | **全角** |
| `10`〜`18` | 4,374〜53,106 | 半角 |
| `止`（中止） | 3,468 | — |
| `外`（除外） | 1,794 | — |
| `消`（取消） | 1,166 | — |
| `①` `②` `④` | 8 / 4 / 1 | 丸数字 |

**したがって `finish_position ~ '^[0-9]+$'` は 10〜18着だけに一致し、
`parseInt('１')` は NaN になる。**

これを踏まえずに実装すると **1〜9着の馬が全員 silently 欠損** し、
最も重要な上位馬が評価対象から消える。

**v2 の対応:** `utils/parse-helpers.ts` の `parseFinishPosition`（全角対応済み）と同等の処理を必須とする。
さらに **中止・除外・取消の 99 を「99着」として使ってはならない**。
`PastRaceSample.finishPosition = null` + `abnormalFinish = true` として欠損扱いする。

## 3. corner_1..4 は「最後のN個のコーナー」を右詰め格納【設計に直結】

埋まり方パターン（全822k行）:

| パターン | 件数 | 該当距離 |
|---|---:|---|
| `--34` | **396,487** | ダ1200 / ダ1400 / 芝1200 / 芝1600 / 芝1400 / ダ1600 |
| `1234` | **368,236** | ダ1800 / 芝2000 / ダ1700 / 芝1800 / 芝2200 / 芝2400 … |
| `-234` | 47,904 | 芝1600 / 芝1800 / 芝2000 |
| `---4` | 4,388 | — |
| `----` | 4,070 | 直線競走等 |

つまり **2コーナーのレースでは corner_3 / corner_4 に値が入り、corner_1 / corner_2 は空**。

**legacy への影響:** `capability-analyzer.ts:82-103` は `lastRace.corner1` を startSpeed の20%に使うが、
`corner_1` の充足率は **44.9%** しかない。1200〜1600m のレースでは常に欠損する。

**v2 の対応:** 「前半の位置取り」は **最初に埋まっているコーナー**（`firstCornerPosition`）を使う。
「直線入口の位置」は `corner_4`（`lastCornerPosition`、充足率99.4%）を使う。

### 妥当性の実測（n=39,684）

| 相関 | 値 |
|---|---:|
| corr(最初コーナー frontRatio, ゴール frontRatio) | **0.2911** |
| corr(4角 frontRatio, ゴール frontRatio) | **0.5001** |

ゴールに近い4角の方が相関が高く、想定どおり。
そして **この 0.29 と 0.50 の差そのものが「道中の維持力」の信号** であり、
3層分離（前半/道中/後半）に十分な情報量があることを裏付ける。

retention（ゴール frontRatio − 最初コーナー frontRatio）の分布:

| p05 | p25 | 中央 | p75 | p95 |
|---:|---:|---:|---:|---:|
| −0.714 | −0.267 | 0.000 | +0.200 | +0.556 |

中央 0 で対称・十分な広がりがあり、維持力モデルの入力として使える。

## 4. umadata に大量の重複行【recency weighting を壊す】

| 項目 | 値 |
|---|---:|
| 総行数 | 822,215 |
| ユニークな (race_id, umaban) | 333,634 |
| 重複しているペア | **27,183** |
| 最大重複数 | **30 行** |
| 1レースあたり行数 | min 5 / 中央 15 / **最大 540** |

重複行は内容が完全同一（同じ馬名・着順・タイム）。

**影響:** 過去走を取得すると同じレースが最大30走として現れ、
recency weighting（前走1.00 / 2走前0.75 …）が1レースに占有される。

**v2 の対応:** `PastRaceSample` 構築時に `(race_id, umaban)` で重複除去し、日付降順で並べることを必須とする。
`lib/race-forecast-v2/types.ts` の `ForecastHorseInputV2.pastRaces` に明記した。

### 監査レポートの訂正: PFS のスケール

PHASE 0 監査（`docs/RACE_FORECAST_AUDIT_PHASE0.md` §0(2)）では、テストフィクスチャの値（1.8〜6.2）から
**「pfs_past は 0〜10 スケール」と結論したが、これは誤りだった。**

実データは **min 3.00 / 中央値 42.33 / max 81.16** で、概ね 0〜100 スケールである。
したがって:

- `capability-analyzer.ts:74` の「PFSは既に0-100スケールと仮定」は **正しい**
- `start-phase.ts:91-94` の閾値（`>=60 → +4`, `>=80 → +8`）は **実データで発火する**（p75=51.66, p99=69.83, max=81.16）
- 欠損時の default 50 は中央値 42.33 より高いため **依然として欠損馬が有利** だが、
  監査で報告した「+8.82 の差」ほど極端ではない。default 50 は実分布の約70パーセンタイル相当。

**訂正後も結論は変わらない:** 欠損に default を「値」として与える設計は欠損馬を有利にする。
ただし「突き抜け」の主因としての比重は、**能力値の全馬100飽和**（監査 §0(1)）の方が大きい。
また `pfs_past` の充足率は **17.4%** しかなく、82.6% の行で欠損 default が使われている。

**バックテスト方針:**
指数（L4F/T2F/potential/makikaeshi/pfs_past）を使う評価は **2024年以降の約8,300レース** が対象。
2019〜2023 は umadata 由来指標（last_3f / pci / corner / margin / field_size）のみで評価可能なので、
「指数なしでも動く」ことの検証に使える。

1頭あたり出走数（recency weight の妥当性根拠）:

| 条件 | 頭数 | 割合 |
|---|---:|---:|
| 1走以上 | 42,689 | 100.0% |
| 2走以上 | 40,120 | 94.0% |
| 3走以上 | 36,413 | 85.3% |
| 4走以上 | 32,151 | 75.3% |
| **5走以上** | 28,310 | **66.3%** |
| 8走以上 | 20,658 | 48.4% |

平均 19.26 走。**直近5走を対象にすると 66.3% の馬が満額信頼度に到達し、85.3% が3走以上**を確保できる。
初期提案の recency weight（1.00 / 0.75 / 0.55 / 0.35 / 0.20）を採用する。
この weight から導かれる信頼度は 1走 0.351 / 2走 0.614 / 3走 0.807 / 4走 0.930 / 5走 1.000。
